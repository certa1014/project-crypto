# 1. Introduction

## 1. Summary

본 프로젝트인 'Project Crypto'는 플레이어가 암호 해독가로서 NPC들의 의뢰를 처리하며 생존해 나가는 1인 플레이 암호 해독 시뮬레이션 게임입니다.

## 2. Introduce

직관적 퍼즐 시스템: 플레이어는 복잡한 수식이나 알고리즘을 계산할 필요 없이 암호 원판, 격자, 투명 필름 등의 시각적이고 물리적인 도구를 직접 조작하며 해독의 직관적인 재미를 느낄 수 있습니다.

도덕적 딜레마와 내러티브: 해독된 결과물을 내용을 임의로 변조하거나 경찰에 신고할 수 있습니다. 이러한 플레이어의 선택은 자원과 세력별 평판에 영향을 미치며 일일정산 플레이에 영향을 끼칩니다.

지속적인 성장 요소: 일일 정산 시스템을 통해 획득한 자금으로 새로운 방식의 암호 해독 기기를 상점에서 구매하여, 퍼즐의 볼륨과 난이도를 점진적으로 확장해 나가는 구조를 갖추고 있습니다.

## 3. Goal

본 Design 보고서에서는 Project Crypto의 Use Case에 대해 Sequence Diagram을 그리고, 각 기능이 어떤 순서로 동작하는지 소개한다.

또한 프로젝트에서 사용되는 주요 클래스들을 Class Diagram으로 표현하고, 각 클래스가 어떤 역할과 기능을 수행하는지 설명한다. 이를 통해 NPC 의뢰 접수, 가이드북 확인, 암호 해독, 결과 제출, 변조, 신고, 일일 정산, 시설 업그레이드가 어떤 객체 구조를 통해 동작하는지 확인할 수 있다.

마지막으로 게임이 완성되었을 때의 State Machine Diagram을 작성하여, 메인 메뉴부터 스테이지 시작, NPC 상호작용, 암호 해독, 결과 처리, 일일 정산, 저장, 시설 업그레이드, 게임 오버까지 Project Crypto가 어떤 상태 흐름을 가지는지 보여준다.

해당 보고서를 읽고 나면 Project Crypto가 어떤 방식으로 설계되었으며, 각 시스템이 전체 게임 루프 안에서 어떻게 연결되는지 이해할 수 있다.
