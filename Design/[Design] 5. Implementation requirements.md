## 5.3 Functional Requirements

### 암호 해독 시스템

* NPC는 플레이어에게 암호문과 키를 전달한다.
* 플레이어는 가이드북을 이용하여 적절한 해독 방법을 확인한다.
* 해독 기기를 조작하여 암호문을 평문으로 복호화한다.
* 시스템은 입력된 결과가 올바른지 검증한다.

### 결과 처리 시스템

* 플레이어는 평문을 NPC에게 전달할 수 있다.
* 플레이어는 평문의 일부 내용을 변조할 수 있다.
* 플레이어는 NPC를 신고할 수 있다.
* 결과에 따라 보상 및 평판 변화가 발생한다.

### 성장 시스템

* 하루 종료 시 일일 정산을 수행한다.
* 플레이어는 획득한 자금을 이용하여 새로운 해독 장비를 구매할 수 있다.
* 구매한 장비는 다음 스테이지 시작 시 작업대에 배치된다.

### 저장 시스템

* 하루 종료 시 자동 저장을 수행한다.
* 플레이어는 저장 데이터를 불러올 수 있다.


## 5.4 Non-Functional Requirements

| 내용                             |
| ------------------------------- |
| 일반 사무용 PC에서도 원활하게 동작해야 한다.      |
| 암호학 지식이 없는 사용자도 쉽게 플레이 가능해야 한다. |
| 저장 데이터 손실 없이 게임 진행 상태를 유지해야 한다. |
| 새로운 암호 방식 및 해독 장비 추가가 가능해야 한다.  |
| 신규 암호 해독 퍼즐을 확장 가능하도록 설계한다.     |
| 텍스트와 이미지의 가독성을 보장해야 한다.         |
